# PROMPT: Use Case Deep Dive

Using the product context and prospect information provided, generate a
comprehensive use case analysis document suitable for sharing internally
or with a technical champion.

## Prospect Fit Summary
- Overall fit score: High / Medium / Low
- Why: [2–3 sentence rationale]
- Top product module for this prospect: [Module name + why]

## Use Case Analysis (Top 3–5)

For each use case, provide:

### Use Case [N]: [Name]

**The Pain (in their words):**
Write this as if the champion is describing it to their CTO. Use their
industry language and reference their specific tools where possible.

**Our Solution:**
- Product module/feature: [Name]
- How it technically solves the pain: [Specific mechanism]
- What they'd need to integrate: [Their stack components involved]

**Business Impact:**
- For the champion (technical): [What gets easier / faster / less painful]
- For the buyer (business): [Cost saved, revenue protected, risk reduced]
- Estimated impact: [Directional number if possible — reference similar customers]

**Proof Point:**
Reference the most relevant customer story from the product context,
even if anonymized.

**Discovery Confirmation Questions:**
2 questions to confirm this use case is a real priority for them right now.

**Deal Risk:**
What could make this use case not land? (wrong persona in room, existing
solution, low urgency, etc.)

---

## Competitive Exposure
For this prospect specifically:
- Which competitor are they most likely evaluating alongside us?
- Where do we win in this scenario?
- Where is the risk?

## Recommended Land & Expand Path
- Land: Which use case / module to start with and why
- Expand: What the natural next purchase looks like in 6–12 months
- Trigger: What event or milestone should prompt the expansion conversation
